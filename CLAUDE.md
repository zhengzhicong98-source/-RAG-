# CLAUDE.md — 法律助手项目规范

本文件在每次会话开始时自动加载,以下规则强制执行。

---

## Strix 修复后的处理流程

每次 Strix 完成漏洞修复后,必须按以下顺序处理:

1. **查看改动**
   运行 `git diff` 查看所有改动。

2. **审核改动合理性**
   检查两点:
   - 没有破坏原有功能
   - 修复逻辑正确

3. **无问题 → 自动提交**
   如果审核通过,自动执行:
   ```bash
   git add -A
   git commit -m "fix(security): [漏洞类型] - Strix auto-fix"
   ```
   commit message 中的 `[漏洞类型]` 需替换为具体类型(如 `XSS`、`SQL 注入`、`路径穿越` 等)。

4. **有问题 → 停下报告**
   如果发现改动有问题:
   - 不要提交
   - 直接告诉用户**具体哪里有疑虑**(指出文件、行号、原因)
   - 等待用户裁决后再操作

---

## 禁止行为

- ❌ Strix 修复后不查 diff 就直接提交
- ❌ 发现改动可疑仍然自动提交
- ❌ commit message 不带漏洞类型(必须能追溯是哪类修复)
