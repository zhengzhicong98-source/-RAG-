# Anonymous File Upload to contracts Storage Bucket

**ID:** vuln-0004
**Severity:** MEDIUM
**Found:** 2026-07-12 09:58:30 UTC
**Target:** https://github.com/zhengzhicong98-source/legal-assistant
**Endpoint:** supabase/migrations/00017_fix_contracts_bucket.sql
**Method:** POST
**CWE:** CWE-862
**CVSS:** 5.3

## Description

The `contracts` storage bucket allows any request (including unauthenticated) to upload files. Migration 00014 originally restricted uploads to authenticated users but migration 00017 reverted this change, citing the need for anonymous upload in the contract review feature. However, this leaves the storage bucket open to anonymous abuse (storage costs, malware hosting, etc.).

## Impact

An unauthenticated attacker can upload arbitrary files to the contracts storage bucket. While the file size and MIME type are validated by the bucket configuration (10MB max, image/* + application/pdf), the attacker can still fill the bucket with unwanted content, consume storage quotas, or use it for hosting malicious files accessed via the public read URL.

## Technical Analysis

Migration 00014 initially added `auth.role() = 'authenticated'` to the contracts bucket INSERT policy. However, migration 00017 reverted this change:

```sql
DROP POLICY IF EXISTS "public_upload_contracts" ON storage.objects;
CREATE POLICY "public_upload_contracts" ON storage.objects
  FOR INSERT WITH CHECK (bucket_id = 'contracts');
```

The commit message states: "原因：合同审查功能支持匿名上传（contract/index.tsx 未使用 useAuth）". This means the codebase has a design tension between needing anonymous uploads and security.

## Proof of Concept

Upload a file to the contracts storage bucket without any authentication token - the request will succeed because the INSERT policy has no auth check.

```
import requests

SUPABASE_URL = "https://your-project.supabase.co"
ANON_KEY = "your-anon-key"

# Anonymous file upload to contracts bucket
response = requests.post(
    f"{SUPABASE_URL}/storage/v1/object/contracts/test.txt",
    headers={
        "apikey": ANON_KEY,
        "Content-Type": "text/plain"
        # No JWT token
    },
    data="test content"
)
print(f"Upload status: {response.status_code}")
```

## Code Analysis

**Location 1:** `supabase/migrations/00017_fix_contracts_bucket.sql` (lines 5-7)
  Anonymous file upload to contracts bucket

  **Suggested Fix:**
```diff
- CREATE POLICY "public_upload_contracts" ON storage.objects
-   FOR INSERT WITH CHECK (bucket_id = 'contracts');
+ -- RLS enforced by Edge Function auth; anon key can initiate uploads via signed URL
+ CREATE POLICY "public_upload_contracts" ON storage.objects
+   FOR INSERT WITH CHECK (bucket_id = 'contracts' AND auth.role() = 'authenticated');
```

## Remediation

Restore the authenticated-only upload policy from migration 00014:

```sql
DROP POLICY IF EXISTS "public_upload_contracts" ON storage.objects;
CREATE POLICY "public_upload_contracts" ON storage.objects
  FOR INSERT WITH CHECK (bucket_id = 'contracts' AND auth.role() = 'authenticated');
```

If anonymous upload is needed for the contract review flow, implement signed URL uploads or route uploads through the authenticated Edge Function instead.
